    Continuous Integration

    Setting up Continuous Integration is an always ongoing process, the base has to be done intially, but the process will always grow and there will always be something new to add/setup/define.

    A CI usually consist of several steps, a code quality checker like (es-lint, scss-lint), a test suite be that Unit Tests, E2E Tests a build tool and then an automatic deployment tool.
    Achieving this goal may bring a variety of tools on scene, most of them consist of extensive configuration and experimentation.

    Unit Tests

    As you may understant number tool to test your React App would be Jest as it has good built-in integration and works pretty well combined, thus we have a poor test coverage but we're looking to improve it and get a better result. Works has to be done. Fast Paced start-ups don't give you the possiblity to write too many tests.

  • JS Frameworks

    React

    Love it. Used it in a set of projects. I do consider that React is not always the way to go. It all depends on the scope and goal of your project.

    Love the Redux approach and the scallability, it requires a lot of boilerplating but on scale manner it usually pays-off.

    A massive advantage of React is the React Native framework. It just brings the mobile development closer to Web Developers, this allows everybody to get an entry point into Mobile Development, which is purely magical.

    Vue

    A really nice and hyped framework. I find it extremly popular now-a-days but I miss a lot of stuff there. And I'm not a big fan of the mixed HTML Angular like. In my opinion it is great for small SPAs but I'd go with React for a large APP.

    jQuery

    I still think it was and is a great library for simple web sites, which don't require a lot of js work. It can't really keep upwith how web apps evolved now-a-days but it's a must have in your tech backpack.

    Google Closure Script

    It's a tool that basically mix jQuery and Modern Web Development. You build components, but you bind them to existing DOM, basically trying to keep up the Single Responsibility Principle yet make everything modular.

    It is a pain to configure it, because it uses a .jar file to compile js, but it does an amazing job.
